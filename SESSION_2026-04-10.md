### 5. supervisord_base.conf - multiple optimizations
Final state of vllm serve command:
```
vllm serve Qwen/Qwen3.5-27B-FP8
    --quantization fp8
    --dtype auto
    --host 0.0.0.0
    --port 8000
    --gpu-memory-utilization 0.85
    --max-model-len 16384
    --kv-cache-dtype auto
    --enable-chunked-prefill
    --max-num-batched-tokens 8192
    --enable-prefix-caching
    --language-model-only
    --speculative-config '{"method":"qwen3_next_mtp","num_speculative_tokens":2}'
```

Key changes from original:
- `--kv-cache-dtype auto` (was fp8 - had uncalibrated scaling factor warnings)
- `--max-model-len 16384` (was 32768 - voice app only needs 150 output tokens)
- `--enable-prefix-caching` (system prompt is constant, big win for multi-turn)
- `--language-model-only` (model is a VLM, skip vision encoder - frees KV cache)
- `--speculative-config qwen3_next_mtp` (MTP speculative decoding, ~1.5-2x decode speedup)

### 7. Replace Deepgram Flux with local VAD+ASR (Silero + Cohere Transcribe)

Research summary:
- **Cohere Transcribe** (`CohereLabs/cohere-transcribe-03-2026`): #1 on Open ASR Leaderboard (5.42% WER), 2B Conformer, Apache 2.0, 524x RTF on GPU. Batch-oriented (VAD-chunked approach).
- **Silero VAD v6** (MIT license): 8kHz native (no resampling from SIP ulaw), 256-sample chunks = 32ms frames, configurable threshold/silence timing.
- **TEN VAD**: Rejected - license has "no competing with Agora" clause despite Apache 2.0 branding.
- **Voxtral Mini 4B Realtime**: Streaming ASR but 480ms per-word delay; streaming advantage wasted in VAD-chunked pipeline.

**Implementation:**

New file: `/xfiles/update_plugins/mr_sip/src/mr_sip/stt/silero_cohere_stt.py`
- `SileroCohereSTT(BaseSTTProvider)` - drop-in replacement for DeepgramFluxSTT
- Silero VAD runs on every 256-byte (32ms) ulaw chunk
- Speech onset fires `turn_resumed_callback` immediately (barge-in detection)
- Buffers ulaw bytes during speech
- On end-of-speech: ulaw -> PCM float32 -> 2x linear resample to 16kHz -> Cohere Transcribe -> emit final
- Transcription runs in executor (non-blocking)
- Hard cap: `max_utterance_duration_s` (default 30s) prevents runaway buffering

Updated files:
- `stt/stt_factory.py` - added `silero_cohere` provider
- `stt/__init__.py` - exported `SileroCohereSTT`
- `services_v2.py` - `is_local_provider` flag bypasses REQUIRE_DEEPGRAM checks; silero_cohere config block reads env vars
- `Dockerfile` - Stage 3c: `pip install silero-vad accelerate` + `transformers>=4.40.0` upgrade

## Model Notes
- **Qwen3.5-27B-FP8** is a VLM (vision-language model) with Gated DeltaNet hybrid architecture
- Thinks by default - caller must pass `chat_template_kwargs: {enable_thinking: false}` per request
- MTP trained, supports `qwen3_next_mtp` speculative decoding
- Tested: ~32-50 tokens/s on H200 (before MTP + language-model-only optimizations)
