## Current State

- **Default TTS backend:** `qwen3tts_openai` (groxaxo on port 8880)
- **Default model:** `Qwen/Qwen3-TTS-12Hz-0.6B-CustomVoice` (built-in voices work, clone: triggers auto-switch to Base model)
- **Voice registration:** Working. `POST /v1/audio/voice-register` downloads audio, auto-transcribes with Whisper, creates ICL mode profile
- **Plugin auto-registration:** When `voice_id` is a URL, plugin auto-registers on first use, caches URL→`clone:Name`

## Known Issues

1. **Audio quality over SIP** - Raw PCM sounds clean but SIP delivery has artifacts. Likely a buffering/pacing issue in the AudioPacer or resampling pipeline. Not debugged yet.
2. **Flash attention not installed** - Falls back to SDPA. Works fine but slightly slower. Pre-built wheels available via `pip install flash-attn --index-url=https://download.pytorch.org/whl/cu124` if needed later.
3. **First clone request slow** - Model switch CustomVoice→Base takes ~10-30s. Subsequent requests are fast.
4. **Whisper model** - Uses `base` by default. Set `WHISPER_MODEL=tiny` for faster transcription or `WHISPER_MODEL=small` for better accuracy.

## Deploy Steps

1. Delete old RunPod template: `python deploy_qwen.py --delete-template`
2. Rebuild Docker image and push
3. Deploy: `python deploy_qwen.py --deploy`
4. Set plugin env: `MR_QWEN3TTS_BACKEND=openai`, `MR_QWEN3TTS_OPENAI_URL=https://<POD_ID>-8880.proxy.runpod.net`
5. Set agent persona `voice_id` to a URL or `clone:Name`

## Voice Cloning Flow

```
Agent persona: {"voice_id": "https://example.com/voice.mp3"}
  ↓ first request
Plugin: POST /v1/audio/voice-register {name, ref_audio_url, x_vector_only_mode: false}
  ↓
Server: download audio → Whisper transcribe → create ICL profile
  ↓
Plugin: cache URL → "clone:voice", then POST /v1/audio/speech {voice: "clone:voice"}
  ↓ all subsequent requests
Plugin: cache hit → skip registration, direct TTS request
```
